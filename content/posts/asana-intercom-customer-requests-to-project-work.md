# Automating Customer Requests to Project Work: A Guide for Asana and Intercom
As a B2B team, integrating Asana and Intercom can significantly improve your customer request handling process. By automating the conversion of tagged Intercom conversations into Asana tasks, you can ensure that customer requests are prioritized and addressed in a timely manner, leading to increased customer satisfaction and improved project delivery.

## Introduction
The integration of Asana and Intercom offers a substantial return on investment (ROI) for B2B teams. By streamlining customer requests and project work, teams can reduce manual effort, enhance collaboration, and ultimately drive business growth.

## Feature Comparison
| Feature | Asana Capability | Intercom Capability |
| :--- | :--- | :--- |
| Task Management | Create and assign tasks, set deadlines, and track progress | Not applicable |
| Customer Communication | Not applicable | Manage customer conversations, send targeted messages, and automate workflows |
| Tagging and Filtering | Tag and filter tasks for easy organization and prioritization | Tag and filter conversations for easy organization and follow-up |
| Integration | Integrate with various tools and platforms, including Intercom | Integrate with various tools and platforms, including Asana |

## Technical Prerequisites
To integrate Asana and Intercom, you will need:
* Asana API access with a valid API key
* Intercom API access with a valid API key
* Webhooks set up in both Asana and Intercom to enable real-time data exchange

## The Workflow
The integration workflow involves the following 5-step logic flow:
1. Intercom conversations are tagged with a specific keyword or phrase.
2. When a conversation is tagged, Intercom triggers a webhook that sends the conversation data to Asana.
3. Asana receives the conversation data and creates a new task with the relevant details.
4. The task is assigned to a team member or project, and deadlines are set as needed.
5. Asana updates are synced back to Intercom, ensuring that customer requests are tracked and addressed.

## Best Practices
To ensure a secure and efficient integration, follow these best practices:
* Use secure API keys and webhooks to protect sensitive data.
* Set up a regular sync frequency to ensure that data is up-to-date and consistent across both platforms.

> [!TIP]
> **Pro-Tip:** Use custom fields in Asana to capture specific details from Intercom conversations, such as customer names or request types, to enhance task organization and filtering.

## FAQ
1. **Q: How do I ensure that only specific Intercom conversations are converted to Asana tasks?**
A: Use Intercom's tagging and filtering features to identify the conversations that should be converted to tasks.
2. **Q: Can I customize the task creation process in Asana based on Intercom conversation data?**
A: Yes, you can use Asana's custom fields and workflows to capture specific details from Intercom conversations and create tasks accordingly.
3. **Q: How often should I sync data between Asana and Intercom?**
A: The sync frequency depends on your team's workflow and requirements. As a best practice, sync data at regular intervals, such as every 15-30 minutes, to ensure that data is up-to-date and consistent across both platforms.
